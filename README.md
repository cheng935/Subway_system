# Subway Pathfinding System

本项目实现了一个基于 A* 算法与广度优先搜索（BFS）的地铁线路路径搜索系统。  
通过构建站点图结构，用户可以输入起点与终点站名，系统将输出对应的最优路径。


## Features

- 使用 A* 算法进行启发式最优路径搜索  
- 使用 BFS 查找无权图的最短路径  
- 利用站点的坐标信息构建启发函数  
- 支持多条地铁线路的建图  
- 控制台交互式输入与输出  


## Algorithms

### A* Algorithm

A* 是一种启发式最优搜索算法，用于在已知图结构中寻找代价最小的路径。其评价函数为：

$$
f(n) = g(n) + h(n)
$$

- g(n)：从起点到当前节点的实际代价  
- h(n)：从当前节点到终点的预估代价  

本项目使用欧几里得直线距离作为 h(n)。

$$
h(n) = \sqrt{(x_2 - x_1)^2 + (y_2 - y_1)^2}
$$

为了保证 A* 的最优性，启发函数必须不高估实际代价（即可低估不可高估）。

A* 相比 Dijkstra 算法具有更强的目标指向性，因此在大多数场景中效率更高。

### BFS

BFS 按层扩展节点，可用于无权图中寻找最短路径。  
因为适用于无权图，BFS找到的实际是经过站点最少的路径而非最近。
其特点是一定能够找到站点最少的路径，但因没有目标方向的引导，会在范围较大的图中扩展大量无关节点。
